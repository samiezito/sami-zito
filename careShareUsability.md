# Care Share Health Alliance: Usability Evaluation & Content Design Recommendations

## Overview
This project evaluates the usability of the Care Share Health Alliance website, a statewide resource designed to help underserved communities in North Carolina access health and social services.

The goal of this evaluation was to identify usability barriers that impact users’ ability to complete key tasks efficiently and to translate those findings into actionable content and documentation improvements.

## Audience
- Community leaders seeking healthcare and community-based resources  
- Users with varying levels of digital literacy  

## Problem
Users rely on the Care Share website to find critical services, but usability issues can prevent them from:
- locating relevant resources  
- understanding available services  
- completing key actions (such as event registration)

Poor navigation, unclear labeling, and insufficient task guidance reduce both task success and user confidence.

## My Role
- Designed and conducted usability testing  
- Developed task scenarios reflecting real user goals  
- Measured task success, user confidence, and time-on-task
- Analyzed findings and identified usability breakdowns  
- Translated results into content and structural recommendations  

## Methods
- Task-based usability testing with 5 participants  

**Metrics collected:**
- Task completion (success/failure)  
- Time-on-task
- Self-reported metrics (e.g., confidence, perceived usability)

**Tasks included:**
- Locating services  
- Navigating site content  
- Completing an event registration  
- Evaluating information credibility  


## Key Findings

### Overall Performance
- **48% task success rate** across all tasks  
- Significant usability barriers impacting core functionality  
- Average time on task was 94 seconds for successful tasks. Users could complete task quickly

### Common Issues
- Unclear navigation pathways  
- Inconsistent or ambiguous labeling  
- Lack of step-by-step guidance for complex actions  
- Difficulty locating key features

## Documentation & Content Implications

This project highlights how usability issues are often **content and structure problems**. Based on my knowledge of documentation and usability I made the following recommendations:

1. Task-Based Documentation
- Create clear, step-by-step instructions for key actions (e.g., event registration)  
- Reduce cognitive load by guiding users through workflows  

2. Improved Labeling & Terminology
- Replace vague or inconsistent labels with user-centered language  
- Align terminology with user expectations  

3. Navigation Restructuring
- Organize content around user goals rather than internal categories  
- Improve discoverability of high-priority tasks  

4. Scannable Content Design
- Use headings, bullet points, and chunked information  
- Support quick information retrieval  

## Outcomes
This evaluation identified critical usability breakdowns that directly impact access to health resources. The findings provide a foundation for:
- Improving task success rates  
- Increasing user confidence  
- Aligning content structure with user needs
